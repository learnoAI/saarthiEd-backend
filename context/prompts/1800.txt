**TASK:**
Analyze the image to extract math problems and student answers. Output the result as a single JSON object containing a list under the key `"extraction_result"`.

**JSON STRUCTURE:**
You must return a valid JSON object matching this exact structure:
```json
{
  "extraction_result": [
    {
      "question_number": 1,
      "question": "Q1. 770 - 86",
      "student_answer": "684"
    },
    ...
  ]
}
```

1.  **`question_number` (Integer):**

    -   **MUST** be an integer data type (e.g., `1`, not `"Q1."` or `"1"`).
    -   Strip all prefixes ("Q") and punctuation (".").

2.  **`question` (String):**
    -   Combine the label and the math problem into a **single line**.
    -   **CRITICAL:** Replace all visual newlines/vertical stacking with a single space.
    -   Format: `"Label. TopNumber - BottomNumber"` (e.g., `"Q6. 590 - 99"`).
    -   Do **NOT** include `\n` characters.

3.  **`student_answer` (String):**
    -   Extract the handwritten number exactly as written.
    -   If the answer space is empty, return `""`.

-   **Digit Confusion:** Pay extreme attention to **1 vs 7**.
    -   A vertical line without a clear top crossbar or sharp angle is a **1**.
    -   Do not confuse "491" for "497" or "591" for "597" unless distinct stroke evidence exists for a 7.
-   **General:** Look for stroke continuity to separate digits from noise.

**SORTING & OUTPUT ORDER:**
-   **CRITICAL:** The output list **MUST** be sorted numerically by `question_number` (1, 2, 3... 40).
-   Do not sort lexicographically (do not put Q10 after Q1).
-   Ensure all visible questions are extracted.

**CONSTRAINT:**
-   Output **ONLY** the JSON object. No markdown formatting, no code blocks, no commentary.
